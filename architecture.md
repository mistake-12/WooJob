# 项目核心指南 (Project Context)

## 1. 角色设定
你是一个顶级全栈工程师，精通 Next.js (App Router)、Tailwind CSS 和 React。你的代码风格严谨、模块化，且极具设计感。

## 2. 业务需求 (PRD - MVP阶段)
- **核心目标：** 为大学生做一个极简的求职看板，管理投递进度。
- **核心功能：** 
  1. 横向状态看板（待投递、已投递、笔试中、面试中、OFFER、已结束）。
  2. 卡片拖拽变更状态。
  3. 卡片包含：公司、岗位、截止时间、状态标签。
  4. 右侧全局侧边栏，用于未来接入 AI 助手对话。

## 3. 技术栈与纪律 (Tech Stack & Rules)
- **框架：** Next.js 14, React, Tailwind CSS。
- **UI 组件库：** shadcn/ui (仅作为底层，样式需覆盖)、Lucide React (图标)。
- **拖拽库：** @hello-pangea/dnd。
- **代码纪律：**
  - 不允许写内联 CSS (style={...})，全部使用 Tailwind class。
  - 遇到复杂组件，必须拆分成单独的文件放到 `components/` 目录下。
  - 优先使用本地 mock 数据进行开发测试。

## 4. UI布局和设计
- 桌面端应用，宽度1280px，高度自适应
- 整体采用大地色系配色：
  - 背景色: #D1CFCA (外围)
  - 主背景: #EBE8E3 (卡片区域)
  - 列背景: #E8E5E0 
  - 强调色: #8B735B (棕色，用于进度条、标签)
  - 文字: #1A1A1A (主文字), #666666 (次要文字)
- 圆角大边框风格，主容器border-radius: 32px
- 字体使用 Inter + JetBrains Mono

### 核心功能：拖拽式看板 (Kanban Board)
6列布局，每列代表求职阶段：
1. 待投递 (To Apply) - 3个卡片
2. 已投递 (Applied) - 1个卡片  
3. 笔试中 (Testing) - 1个卡片
4. 面试中 (Interview) - 2个卡片
5. Offer - 0个卡片
6. 已结束 (Rejected) - 12个卡片

每列顶部显示：阶段名称 + 卡片数量(如"(03)")

### 求职卡片设计
每个卡片固定高度180px，包含：
- 序号 (如"(01)")
- 职位名称（大字粗体）：如"产品经理"、"高级产品经理"、"UI设计师"等
- 公司名称（棕色）：字节跳动、阿里巴巴、腾讯、美团、滴滴出行
- 元信息行（两列）：
  - 内推: 有/无/学长
  - 截止: 3天后/5天后/2天前
  - 剩余: 24小时
  - 轮次: 技术二面
  - 时间: 明天14:00
- 底部进度条：
  - 细条高度4px，棕色填充
  - 根据列自动调整进度百分比：
    - 待投递: 5%
    - 已投递: 25%  
    - 笔试中: 50%
    - 面试中: 75%
    - Offer: 100%
  - 进度条下方两个标签（如"草稿→录用"）

卡片样式：
- 白色背景 #FAF9F7
- 1px边框 rgba(139, 115, 91, 0.2)
- 圆角8px
- 右上角有三条横线作为拖拽手柄
- hover时阴影加深、上浮2px

### 拖拽交互功能（必须实现）
- 卡片可拖拽(draggable=true)
- 拖拽时卡片放大1.02倍、旋转2度、半透明
- 拖放到目标列时，该列背景高亮（虚线边框）
- 放置后：
  - 进度条自动更新为新列对应百分比
  - 列计数自动更新
  - 放置动画（弹跳效果）
- 底部显示提示"拖动卡片到目标列以更新状态"
