## 一、产品概述

### 1.1 背景与机会

互联网求职者在求职过程中面临以下核心问题：

1. **能力与岗位不匹配**：想投 AI 产品经理但缺乏 AI 项目经历，不知道该补什么
2. **准备碎片化**：简历优化、面试练习、投递策略各自为战，缺乏系统性
3. **反馈缺失**：练了很多面试题，但不知道自己到底差在哪
4. **全程无陪伴**：从投简历到拿 offer 的漫长过程中，没有人指导

现有竞品（牛客网、力扣、ChatGPT）只覆盖了"面试练习"这一个环节，且缺乏个性化和持续陪伴。

### 1.2 产品定义

```
For 正在求职的互联网从业者（0-5 年经验）
that need 系统性的求职指导和持续陪伴
WooJob is a AI 求职陪跑教练
that 从能力诊断到面试模拟到 Offer 谈判，提供全程个性化指导
Unlike 牛客网只提供题库、猎聘只提供职位信息
WooJob provides 像一个资深猎头一样陪你走完求职全程
```

### 1.3 目标用户

| 用户画像 | 特征 | 核心需求 | 优先级 |
|----------|------|----------|--------|
| **应届求职者** | 0-1 年经验，校招，焦虑感强 | 系统性指导，不知道从哪开始 | P0 |
| **跳槽者（初级）** | 1-3 年经验，想换公司/方向 | 能力诊断 + 面试准备 | P0 |
| **转行者** | 非互联网转互联网，或跨岗位 | 差距填补 + 学习路径规划 | P0 |
| **跳槽者（中级）** | 3-5 年经验，有明确目标 | 面试精进 + Offer 谈判 | P1 |

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 用户自评准备充分度 | 使用前 4/10 → 使用后 7/10 | 用户调研 |
| 面试通过率 | 提升 30%（vs 用户历史） | 用户自报 |
| 7 日留存率 | > 40% | 埋点数据 |
| 完整旅程完成率 | > 25% | 埋点数据 |
| NPS | > 40 | 用户调研 |

## 二、产品架构

### 2.1 用户旅程

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│  能力   │──→│  差距   │──→│  简历   │──→│  面试   │──→│  投递   │──→│  Offer  │
│  诊断   │   │  填补   │   │  优化   │   │  准备   │   │  策略   │   │  谈判   │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
   2-5min       1-4 周        15-30min      持续练习       15-30min      15-30min
```

### 2.2 Agent 架构

```
┌─────────────────────────────────────────────────────────┐
│                   Director Agent                        │
│  - 解析用户输入，路由到对应阶段                           │
│  - 管理全局状态和阶段切换                                 │
│  - 协调子 Agent 之间的上下文传递                          │
└───────┬───────────┬───────────┬───────────┬─────────────┘
        │           │           │           │
        ▼           ▼           ▼           ▼
┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐
│  Career    │ │  Learning  │ │  Resume    │ │ Interviewer│
│  Coach     │ │  Advisor   │ │  Agent     │ │ + Assessor │
│ 能力诊断   │ │ 差距填补   │ │ 简历优化   │ │ 面试模拟   │
└────────────┘ └────────────┘ └────────────┘ └────────────┘
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
┌────────────┐          ┌────────────┐
│  Strategy  │          │   Offer    │
│  Agent     │          │   Agent    │
│ 投递策略   │          │ Offer谈判  │
└────────────┘          └────────────┘
```

### 2.3 数据流

```
用户输入 → Director Agent（路由） → 阶段 Agent（处理） → LLM（生成）
                ↓                        ↓
          阶段状态更新              产出物存储
                ↓                        ↓
          前端状态同步              侧边栏展示
```

## 三、功能需求

### 3.1 P0 功能（MVP 必须）

#### F-001：能力诊断

**用户故事**：作为求职者，我想知道我的能力和目标岗位的差距，以便有针对性地准备。

| 项目 | 说明 |
|------|------|
| **触发条件** | 用户首次进入，或手动触发"能力诊断" |
| **输入** | 目标岗位（文本/选择）、当前简历（上传/粘贴）、自我描述（可选） |
| **处理** | 1. 解析岗位 JD 提取能力要求<br>2. 解析简历提取当前能力<br>3. 计算差距并排序 |
| **输出** | 能力雷达图、差距清单（高/中/低优先级）、整体匹配度百分比 |
| **验收标准** | 用户上传简历 + 输入岗位后，10 秒内生成诊断报告 |

**差距维度**：

| 维度 | 说明 | 示例 |
|------|------|------|
| 知识类 | 理论知识、方法论 | "了解 AI 产品生命周期" |
| 经验类 | 项目经历、实战经验 | "有 AI 项目落地经验" |
| 工具类 | 具体工具使用 | "熟练使用 Figma" |
| 软技能 | 沟通、协作、领导力 | "跨部门协作能力" |

---

#### F-002：差距填补与学习计划

**用户故事**：作为求职者，我想获得针对能力差距的学习计划，以便高效补齐短板。

| 项目 | 说明 |
|------|------|
| **触发条件** | 能力诊断完成后，用户点击"制定计划" |
| **输入** | 差距清单（来自 F-001）、每日可用时间、学习偏好（看视频/读书/做项目） |
| **处理** | 1. 为每个高优先级差距生成学习项<br>2. 排序：知识 → 练习 → 项目<br>3. 估算总时间 |
| **输出** | 学习计划列表（含资源推荐）、预计完成时间、进度追踪 |
| **验收标准** | 计划包含至少 1 个项目实战建议，且每个学习项有具体资源 |

**计划项类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 学习 | 知识获取 | "阅读《AI 产品经理：人工智能时代的产品设计》" |
| 练习 | 技能练习 | "用 ChatGPT API 搭建一个简单的对话 demo" |
| 项目 | 实战项目 | "从 0 到 1 设计一个 AI 客服产品方案" |
| 资源 | 参考材料 | "推荐课程：吴恩达 AI for Everyone" |

**与简历优化的衔接**：
- 完成项目类计划项后，系统提示"把这段经历加到简历"
- 点击后跳转到简历优化阶段，预填新经历

---

#### F-003：简历优化

**用户故事**：作为求职者，我想针对目标岗位优化简历，以便提高简历通过率。

| 项目 | 说明 |
|------|------|
| **触发条件** | 用户手动进入，或从差距填补阶段跳转 |
| **输入** | 当前简历（上传/粘贴）、目标岗位、差距填补阶段的新经历（可选） |
| **处理** | 1. 分析简历与岗位匹配度<br>2. 识别可优化的表述<br>3. 针对性改写建议 |
| **输出** | 优化建议列表、改写后的措辞、匹配度提升预估 |
| **验收标准** | 至少给出 3 条具体可执行的优化建议 |

---

#### F-004：面试模拟

**用户故事**：作为求职者，我想进行模拟面试并获得反馈，以便提升面试表现。

| 项目 | 说明 |
|------|------|
| **触发条件** | 用户选择"面试准备"阶段，选择目标岗位和公司 |
| **输入** | 目标岗位、目标公司（可选）、简历、面试轮次（一面/二面/HR 面） |
| **处理** | 1. 生成针对性问题<br>2. 用户回答后追问 1-2 轮<br>3. 每轮评估回答质量<br>4. 面试结束后生成报告 |
| **输出** | 模拟面试对话、实时回答提示、每轮评分、面试报告 |
| **验收标准** | 至少支持 5 轮问答，每轮有追问，结束后有结构化反馈 |

**追问逻辑**：

| 用户回答质量 | 追问策略 |
|-------------|----------|
| 缺乏数据 | "能给个具体数字吗？" |
| 缺乏细节 | "你在其中具体做了什么？" |
| 缺乏逻辑 | "你为什么选择这个方案？" |
| 回答良好 | "如果重来一次，你会做什么不同的决定？" |

**评分维度**：

| 维度 | 权重（默认） | 说明 |
|------|-------------|------|
| 准确性 | 20% | 回答是否正确 |
| 深度 | 25% | 是否有深入分析 |
| 逻辑性 | 20% | 思路是否清晰 |
| 完整性 | 20% | 是否覆盖要点 |
| 创新性 | 15% | 是否有独到见解 |

## 六、交互设计概述

### 6.1 核心交互理念

**"一个对话流，多个 Agent，一条时间线"**

- 顶部时间线：进度导航，可点击跳转
- 主对话区：单一连续对话流，Agent 随阶段自动切换
- 侧边栏：阶段产出物、评分、快捷操作

### 6.2 页面结构

```
┌──────────────────────────────────────────────────────┐
│  Header                                              │
├──────────────────────────────────────────────────────┤
│  ProgressTimeline: ✅诊断 → ✅填补 → 🔵简历 → ○面试  │
├──────────────────────────┬───────────────────────────┤
│                          │  Sidebar                  │
│  ChatContainer           │  - 阶段信息               │
│  - MessageBubble         │  - 产出物                 │
│  - TipCard               │  - 评分                   │
│  - GapAnalysisCard       │  - 快捷操作               │
│  - PlanCard              │                           │
│                          │                           │
├──────────────────────────┴───────────────────────────┤
│  InputArea: [文本] + [语音] + [上传] + [发送]        │
└──────────────────────────────────────────────────────┘
```

### 6.3 关键交互流程

**能力诊断 → 差距填补流程**：

```
1. 用户输入目标岗位（如"AI 产品经理"）
2. 用户上传当前简历
3. Agent 生成能力诊断报告（GapAnalysisCard）
   - 雷达图展示各维度能力
   - 高优先级差距列表
   - 整体匹配度 65%
4. 用户点击"制定学习计划"
5. Agent 询问：每日可用时间？偏好学习方式？
6. Agent 生成学习计划（PlanCard）
   - 学习项：阅读 AI 产品相关书籍
   - 练习项：用 ChatGPT API 搭建 demo
   - 项目项：设计一个 AI 客服产品方案
7. 用户逐项完成，进度自动更新
8. 完成项目项后，系统提示"把新经历加到简历"
9. 点击后跳转简历优化阶段，预填新经历
```

### 7.3 能力诊断

```typescript
interface GapAnalysis {
  id: string;

interface SkillGap {
  skill: string;
  category: 'knowledge' | 'experience' | 'tool' | 'soft_skill';
```

## 九、风险与应对

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| LLM 回答质量不稳定 | 高 | 中 | 多轮 prompt 优化 + 人工审核样本 |
| 用户中途流失 | 高 | 高 | 阶段产出物即时可见 + 进度激励 |
| 评分准确性不足 | 中 | 中 | 初期定性反馈为主，逐步引入定量评分 |
| 学习计划过于泛化 | 中 | 中 | 基于用户具体差距生成，非通用模板 |
| 面试题库冷启动 | 中 | 低 | 先用 LLM 生成 + 公开数据集 |
| 成本过高 | 中 | 中 | 缓存上下文 + 选择性使用 GPT-4o |
