个人中心与账号体系 (User Profile & Account)
一、 需求概述
为系统增加统一的“个人中心”模块。作为用户资产的“大本营”，不仅提供基础的账号管理（登出），更核心的业务价值是存放用户的“默认简历 (PDF)”和“求职偏好”，为后续的大模型 AI 模拟面试提供 Context (上下文数据)。
二、 前端 UI/UX 设计
1. 入口设计 (The Entry)
位置： 右上角 Header 区域，完全替换掉现有的“状态：求职中”模块。
UI 展现： 一个圆形的浅色头像框（提取用户邮箱的首字母，如“W”），旁边配一个极其低调的小箭头 v。
交互 (Dropdown)： 点击头像，弹出一个极简的下拉菜单 (Popover/Dropdown Menu)，包含两个选项：
⚙️ 个人设置 (Settings)
🚪 退出登录 (Log out)
2. 设置面板 (Settings Modal)
为了不干扰主看板，不采用抽屉，采用居中弹窗 (Modal/Dialog)。
布局风格： 纯白卡片，两列布局（左侧导航，右侧表单）。
模块 A：基础信息 (Basic Info)
昵称 (Nickname)
目标岗位 (Target Role，如：高级产品经理) - 用于 AI 推荐的默认提示词。
模块 B：简历库 (Resume Vault) —— 最核心功能
允许用户上传一份 PDF 简历。上传后显示绿色的“已就绪”，这个文件将自动送给 AI Agent 作为模拟面试的底座。
三、 后端架构规划 (Supabase)
1. 数据库结构 (profiles 表)
由于 Supabase 的账号密码是存在加密的 auth.users 表里我们改不了，我们需要建一个关联的 profiles 表来存业务数据。
id (UUID, Primary Key, Foreign Key -> auth.users.id)
email (String)
nickname (String)
target_role (String)
resume_url (String) - 指向 Supabase Storage 的链接
2. Server Actions 接口 (actions/profile.ts)
getProfileAction(): 获取当前登录用户的配置信息。
updateProfileAction(data): 更新昵称、目标岗位等。
signOutAction(): 清除 Session Cookie，重定向回 /login。
