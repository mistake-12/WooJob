## 五、关键发现

### 5.1 WooJob 的差异化机会

| 发现 | 说明 | WooJob 对应策略 |
|------|------|-----------------|
| **全流程空白** | 没有一个产品覆盖从能力诊断到 Offer 谈判的全流程 | 6 阶段完整旅程 |
| **中国市场缺 AI 教练** | 牛客/猎聘/Boss 都是"平台+AI 功能"，不是"AI 教练" | AI-first 产品定位 |
| **能力诊断无人做** | 没有竞品做"你和目标岗位差什么"的结构化分析 | GapAnalysis + 学习计划 |
| **追问训练是盲区** | 面试模拟多为"问-答-下一题"，缺乏深度追问 | 多 Agent 追问逻辑 |
| **阶段割裂** | 用户用 A 工具优化简历、B 工具练面试、C 工具查投递 | 统一时间线+上下文穿透 |

### 5.2 直接竞品 vs 间接竞品

**直接竞品**（功能重叠 > 50%）：
- 牛客网（面试模拟部分）
- 面试鸭（面试模拟部分）
- Teal（如果做中文版）

**间接竞品**（替代行为）：
- ChatGPT/Claude（自由对话式求职辅导）
- 猎头/职业顾问（真人服务）
- 小红书/脉脉面经帖（社区经验）

**最大威胁**：ChatGPT/Claude — 免费、灵活、用户已有习惯。WooJob 必须在**结构化**和**个性化**上显著优于通用 LLM。

### 5.3 竞品未覆盖的用户需求

| 需求 | 当前解决方案 | WooJob 机会 |
|------|-------------|-------------|
| "我不知道该投什么岗位" | 问朋友/看帖子 | 结构化能力诊断 |
| "我缺 AI 项目经历，不知道怎么补" | 自己摸索/报班 | 个性化学习计划 |
| "我练了很多但不知道差在哪" | 找人 mock（难约） | 追问式 AI 面试 + 反馈 |
| "拿到 offer 不知道该不该接" | 问朋友/看薪资帖子 | Offer 评估 + 市场对比 |
| "准备过程太碎片化了" | 多个工具来回切 | 统一时间线贯穿全程 |

## 六、WooJob 的竞争优势总结

```
┌─────────────────────────────────────────────────────────┐
│                  WooJob 核心竞争力                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 全流程覆盖（唯一）                                   │
│     能力诊断 → 差距填补 → 简历 → 面试 → 投递 → Offer     │
│     竞品最多覆盖 1-2 个环节                              │
│                                                         │
│  2. AI-first（中国市场唯一）                             │
│     不是"平台加 AI 功能"，而是"AI 教练"                  │
│     对标 BetterUp 的 AI+教练模式，但面向个人              │
│                                                         │
│  3. 能力诊断 + 差距填补（独家）                          │
│     没有竞品做这件事                                     │
│     用户最大痛点"不知道该补什么"被完全忽略               │
│                                                         │
│  4. 多 Agent 追问（技术壁垒）                            │
│     不是"问-答-下一题"，而是深度追问                     │
│     模拟真实面试中面试官的追问行为                       │
│                                                         │
│  5. 阶段上下文穿透（体验壁垒）                           │
│     面试 Agent 能引用简历 Agent 的结论                   │
│     简历 Agent 能引用差距填补的新经历                    │
│     竞品各环节完全割裂                                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 七、风险提示

| 风险 | 说明 | 应对 |
|------|------|------|
| ChatGPT 替代 | 用户可能觉得"用 ChatGPT 就够了" | 强调结构化、个性化、全流程 |
| 牛客网追赶 | 牛客网有用户基础，可快速加 AI 功能 | 深耕"AI 教练"定位，不与题库竞争 |
| Teal 进入中国 | Teal 是最接近的竞品，可能做中文版 | 先发优势 + 本地化深耕 |
| 用户付费意愿 | 中国市场用户对 AI 工具付费意愿低 | 基础免费 + 高级付费 |
| LLM 成本 | 多 Agent 架构 token 消耗大 | 缓存上下文 + 分级模型 |
