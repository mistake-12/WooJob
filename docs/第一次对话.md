### 2. UI 修复：日程视图标签胶囊样式不统一

**问题:** AgendaView 中"待办事项"标签与 TaskDetails 详情面板中的标签样式不一致。

| 组件 | 圆角 | Padding | tagColors |
|------|------|---------|-----------|
| TaskDetails | `rounded-full` | `px-3 py-1` | 完整 |
| AgendaView | `rounded-md` | `px-2.5 py-1` | 缺少 `待办事项` |

**修复操作：**

1. 统一圆角为 `rounded-full`
2. 统一 padding 为 `px-3 py-1`
3. 补充 `tagColors` 中缺失的 `待办事项` 映射

**第一次改错了背景色为白色**，用户反馈后改为保留原米色 `bg-[#EBE8E1]`：

```tsx
// AgendaView.tsx — 最终正确版本
const tagColors: Record<string, string> = {
  '面试': 'bg-[#EBE8E1] text-[#666666]',
  '笔试': 'bg-[#EBE8E1] text-[#666666]',
  '待投递': 'bg-[#EBE8E1] text-[#666666]',
  '待办事项': 'bg-[#EBE8E1] text-[#666666]',

```

## 二、PRD 核心理解

### 产品定位

- **WooJob!!!** — 面向高校毕业生与年轻白领的 All-in-One AI 求职全生命周期管理平台
- 设计风格：极简社论风（Editorial Design）
- 核心理念：状态驱动视图 + AI 智能提效

### 四大核心痛点与解决方案

| 痛点 | 解决方案 |
|------|----------|
| 状态流转与死线焦虑 | 双视图引擎（看板 + 日程） |
| 信息录入繁琐 | AI 工作流：截图/文本解析一键录入 |
| 岗位搜寻耗时 | 异步爬取 + RAG 智能推荐 |
| 面试无侧重点 | 简历+JD 双向模拟面试 |

### 三大 AI Agent 工作流

1. **智能岗位发现与推送** — Cron Job 爬取 + 向量检索 + 一键入库
2. **定制化模拟面试** — JD + Resume → 多轮对话 → 复盘报告
3. **NLU 极速建档** — 截图/文本解析 → Entity Extraction → 双重入库

## 三、后端开发规划

### P0 阶段（MVP 前后端联调）

- [ ] **Supabase 项目初始化**
  - 建立 `Users`、`Jobs`、`Tasks` 核心表
  - 配置 RLS 行级安全策略
  - 配置 Auth（邮箱/第三方登录）
- [ ] **看板 API**
  - `GET /api/v1/jobs` — 拉取用户看板数据
  - `PATCH /api/v1/jobs/{id}/stage` — 拖拽松手瞬间触发
  - `POST /api/v1/jobs` — 新建岗位卡片
  - `PUT /api/v1/jobs/{id}` — 编辑岗位详情（抽屉保存）
  - `DELETE /api/v1/jobs/{id}` — 软删除至回收站
- [ ] **回收站 API**
  - `GET /api/v1/jobs/trash` — 获取已删除岗位
  - `POST /api/v1/jobs/{id}/restore` — 恢复岗位
  - `DELETE /api/v1/jobs/{id}/permanent` — 永久删除
- [ ] **日程 API**
  - `GET /api/v1/tasks?month=YYYY-MM` — 按月预加载
  - `PATCH /api/v1/tasks/{id}` — 更新待办状态/内容
  - `POST /api/v1/tasks` — 新建日程

### P1 阶段（Alpha — AI Agent 接入）

- [ ] **Agent 工作流三：NLU 极速建档**（优先级最高）
  - 接入 DeepSeek API（或 OpenAI）
  - 实现 Entity Extraction（公司、时间、轮次、链接等字段抽取）
  - 构建预览确认卡片 + 双重入库（Jobs + Tasks）
  - 支持文本解析（邮件/聊天输入）和图片解析（截图 OCR）
- [ ] **向量数据库集成**
  - Supabase pgvector 或 Pinecone
  - 岗位向量 embedding 存储

### P2 阶段（商业化版本）

- [ ] **Agent 工作流一：智能岗位发现**
  - 后端 Cron Job 异步爬取大厂官网
  - RAG 检索链路（用户意图 → 向量检索 → 推荐卡片）
- [ ] **Agent 工作流二：模拟面试**
  - Context 组合（JD + Resume PDF 解析）
  - 多轮对话上下文管理（保持对话窗口）
  - 复盘报告生成与存储（notes 字段）
- [ ] **推送通知体系**
  - 微信公众号 / 飞书 Webhook 提醒

## 五、架构决策备忘

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 前端框架 | Next.js 14 + React | App Router + Server Components 预备 |
| 样式方案 | Tailwind CSS | 快速迭代 + 与设计系统解耦 |
| 状态管理 | Zustand | 轻量、支持 SSR、DevTools 友好 |
| 后端运行时 | Node.js / Serverless | Vercel/Supabase Edge Functions |
| 数据库 | Supabase PostgreSQL | 内置 Auth + RLS + pgvector |
| AI 模型 | DeepSeek / GPT-4o | 成本与能力平衡 |
| 向量库 | Supabase pgvector | 统一数据库，减少 infra 复杂度 |
